# 蓝湖标注面板 DOM 抓取方案

> 目标：从蓝湖网页端标注模式（detailDetach）右侧面板获取精确的 iOS 开发标注值。
>
> 方案：用 AppleScript 直接读取已打开 Chrome 标签页的 DOM，绕过登录、Cookie、API 等复杂环节。
>
> 适用人群：任何需要从蓝湖提取 UI 标注数据的开发者 / 设计还原工程师。

---

## 一、核心原理

### 1.1 问题背景

蓝湖（lanhuapp.com）提供"标注模式"（detailDetach 页面），设计师上传设计稿后，开发者点击任意控件，右侧面板会显示：

- 位置（x, y）
- 大小（width, height）
- 字号、行高、字间距、字体、字重
- 颜色（HEX / RGBA）
- 渐变（多色）
- 圆角、阴影
- iOS / Android / Web / 微信小程序 各平台的尺寸

**这些值就是开发时直接使用的值。**

### 1.2 为什么不用 API？

蓝湖有 API 可以拿到 Sketch JSON 和 Schema JSON，但：

- API 返回的是**原始设计数据**，不是标注面板里**整理后展示给开发者的值**
- 两者有差异（例如字号需要换算、Canvas 尺寸可能对不上）
- Cookie 容易失效，API 请求需要维护登录态

### 1.3 为什么用 AppleScript？

蓝湖标注页是一个 SPA（单页应用），页面结构全部在浏览器内存里。

AppleScript 可以直接向 Chrome 发送 JavaScript 命令，读取**当前打开标签页**的 DOM 内容。条件是：

- 蓝湖标注页已经在 Chrome 里打开
- 用户已经手动登录过
- Chrome 已开启"允许 Apple 事件中的 JavaScript"

这样就不需要任何 Cookie、API Key 或服务器。

### 1.4 数据流向

```
用户打开 Chrome 蓝湖标注页（已登录）
    ↓
手动选中一个控件（右侧面板展开）
    ↓
运行 Python 脚本
    ↓
Python 调用 AppleScript，向 Chrome 执行 JavaScript
    ↓
JavaScript 读取页面 DOM → innerText 文本
    ↓
AppleScript 把文本返回给 Python
    ↓
Python 用正则解析文本，提取所有标注字段
    ↓
输出到终端 + /tmp/lanhu_panel_data.json
```

## 四、脚本内部执行流程（详细拆解）

### 4.1 AppleScript 如何工作

macOS 内置 AppleScript，Python 通过 `subprocess` 调用它。

核心命令结构：

```applescript
tell application "Google Chrome"
    repeat with w in windows
        repeat with t in tabs of w
            if (URL of t) contains "detailDetach" then
                set jsResult to execute t javascript "你的JS代码"
                return jsResult
            end if
        end repeat
    end repeat
end tell
```

翻译成人话：
- "告诉 Chrome，遍历所有窗口"
- "每个窗口里遍历所有标签页"
- "找到 URL 里含 `detailDetach` 的标签页"
- "在这个标签页里执行 JavaScript 代码"
- "把 JavaScript 返回的结果还给我"

### 4.2 JavaScript 做了什么

蓝湖标注面板的内容在页面的 `document.body.innerText` 里。

为了只取标注相关的区域，用 JavaScript 做了区域截取：

```javascript
var t = document.body ? document.body.innerText : '';
var end = t.indexOf('历史版本');
if (end === -1) end = t.length;
return t.slice(0, end);
```

这行代码的意思是：从页面开头到"历史版本"之前的文字都拿回来，这段文字里包含了：

- 顶部平台选择器（iOS / Android / Web / 微信小程序）
- Canvas 尺寸（750px × 1624px / iOS @2x）
- 样式信息（图层名 / 位置 / 大小 / 不透明度）
- 文本信息（字体 / 字号 / 行高 / 颜色 / 渐变）
- 代码区（Objective-C / Swift 代码片段）

### 4.3 为什么不用 JSON 返回而用纯文本

最初尝试用 JSON 返回，但 Chrome AppleScript 里 `execute javascript` 的返回值会被**页面 CSS** 污染，导致 JSON 解析失败。

原因：蓝湖页面内部有大量 CSS 文本（含 `{}`、引号等 JSON 非法字符），AppleScript 把这些也包含在返回值里，导致 JSON 断裂。

解决：直接返回**纯文本**（innerText），用 Python 正则解析，避免 JSON 被污染。

### 4.4 正则解析逻辑

拿到纯文本后，用正则提取每个字段。下面逐个解释：

#### Canvas 尺寸

```python
# 设计稿像素尺寸
m = re.search(r'([0-9]+)px\s*[x×]\s*([0-9]+)px', panel_text)
# 例如: "750px × 1624px" → {'width': '750px', 'height': '1624px'}

# iOS 缩放比例
m = re.search(r'(iOS|Android|Web|微信小程序)\s+@([0-9.]+)x', panel_text)
# 例如: "iOS @2x" → device_scale: '@2x'

# iOS pt 尺寸（根据 scale 计算）
# 750px @2x → 375pt
```

#### 样式信息

```python
# 图层名（"图层" 关键字后面紧跟的文字）
m = re.search(r'图层\s*\n+([^\n]+?)\n+位置', panel_text)

# 位置（两个数字紧跟 "位置"）
m = re.search(r'位置\s+([0-9.]+)pt\s+([0-9.]+)pt', panel_text)
# 例如: "位置\n115pt\n151.5pt" → {'x': '115pt', 'y': '151.5pt'}

# 大小
m = re.search(r'大小\s+([0-9.]+)pt\s+([0-9.]+)pt', panel_text)
```

#### 文本信息

```python
# 字体
m = re.search(r'字体\s+([^\n]+)', panel_text)
# 例如: "字体\n苹方-简 常规体" → '苹方-简 常规体'

# 字号 / 行高 / 字间距
m = re.search(r'字号\s+([0-9.]+)pt', panel_text)
m = re.search(r'行高\s+([0-9.]+)pt', panel_text)
m = re.search(r'字间距\s+([0-9.]+)pt', panel_text)
```

#### 颜色 & 渐变

这是最复杂的部分。蓝湖的颜色信息格式有两种：

**格式一：简单渐变（文字渐变）**

蓝湖标注面板会显示：
- "颜色样式" 块 → 主色（如 `#00CBE0`）
- "颜色" 块 → 两个 HEX 色值（渐变起点/终点）

```
颜色样式
主色
字体
苹方-简 中粗体
...
字号  15pt
...
#00CBE0 100%
HEX
HEX#00CBE0
...
#BEFBFF 100%
HEX
HEX#BEFBFF
#58DEE7 100%
HEX
```

提取逻辑：检测"颜色样式" + "主色"关键字，同时收集所有 HEX 值，前两个不同色值即为渐变。

**格式二：多色渐变（背景/形状）**

蓝湖面板颜色区域会显示多个 HEX + 百分比组合：

```
颜色
#2D2C2C 0%
HEX
HEX#2D2C2C
#2C2C2C 52%
HEX
HEX#2C2C2C
#2A2A2B 81%
HEX
HEX#2A2A2B
...
```

提取逻辑：
```python
gradient_stops = re.findall(r'#([0-9A-Fa-f]{6})\s+(\d+)%', panel_text)
# 匹配 "#2D2C2C 52%" → ('2D2C2C', '52')
```

注意：百分比是**透明度/Alpha 值**，不是渐变位置。

#### 代码区语言检测

```python
for lang in ['Objective-C', 'Swift', 'Java', 'Kotlin', 'XML', 'CSS', 'Python']:
    if lang in code_text:
        result['code_lang'] = lang
        break
```

只要"代码"区域文本里包含语言名称，就认为当前选中的是该语言。

## 六、已知限制

### 6.1 需要手动操作

- 每次换控件都要重新运行脚本（不能实时监听变化）
- Chrome 需要保持打开状态

### 6.2 阴影和边距

正则已写好，但需要实际选中带这些属性的元素来验证。

### 6.3 平台切换

脚本目前通过文本推断当前平台（从 `iOS @2x` 关键字判断）。如果用户切换了平台（例如选 Android），脚本会更新 `current_platform` 字段，但 Canvas pt 换算仍以 `iOS @2x` 为准。

### 6.4 Chrome AppleScript 字符限制

直接用 AppleScript 返回中文文本时，有时会被截断（特别是在返回值含 CSS 时）。当前方案通过返回纯文本 + Python 正则解析规避了这个问题。

## 九、核心代码片段（可直接复用）

### AppleScript + Chrome 执行 JS

```python
import subprocess

js_code = """
(function(){
    var t = document.body ? document.body.innerText : '';
    var end = t.indexOf('历史版本');
    if (end === -1) end = t.length;
    return t.slice(0, end);
})()
"""

escaped = js_code.replace('\\', '\\\\').replace('"', '\\"')
script = f'''
tell application "Google Chrome"
    repeat with w in windows
        repeat with t in tabs of w
            if (URL of t) contains "detailDetach" then
                set jsResult to execute t javascript "{escaped}"
                return jsResult
            end if
        end repeat
    end repeat
    return "NOT_FOUND"
end tell
'''

result = subprocess.run(
    ['osascript', '-'],
    input=script.encode('utf-8'),
    capture_output=True
)
raw = result.stdout.decode('utf-8', errors='replace').strip()
```
