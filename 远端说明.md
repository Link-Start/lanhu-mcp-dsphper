# 远端说明

这个仓库当前保留 3 个有效远端，用途已经明确固定：

- `upstream`：真正的原始源码仓库，只负责同步官方更新
- `github-fork`：用户自己的 GitHub fork，用于备份、协作、PR
- `gitee-fork`：用户自己的 Gitee fork，用于国内备份和日常推送

## 当前远端用途

### 1. `upstream`

```text
https://github.com/dsphper/lanhu-mcp.git
```

用途：

- 唯一的官方上游来源
- 本地 `main` 必须严格对齐这里的 `main`
- 当前本地配置已经禁用向这个远端 `push`

### 2. `github-fork`

```text
https://github.com/Link-Start/lanhu-mcp-dsphper.git
```

用途：

- 用户自己的 GitHub fork
- 适合备份 `main` 和 `lanhu-mcp-ios`
- 如果远端历史与本地定制分支分叉，先确认本地分支才是权威状态，再使用 `--force-with-lease`

### 3. `gitee-fork`

```text
https://gitee.com/Link413/lanhu-mcp-iOS.git
```

用途：

- 用户自己的 Gitee fork
- 当前默认 `push` 目标更偏向这里
- 适合保存最新的 `main` 镜像和 `lanhu-mcp-ios` 定制成果

## 当前分支关系

### `main`

- 本地官方镜像分支
- 必须保持与 `upstream/main` 一致
- 当前跟踪：

```text
upstream/main
```

说明：

- 不在 `main` 上做自定义开发
- 只用来接收上游更新，并在需要时镜像到自己的 fork

### `lanhu-mcp-ios`

- 当前主要工作分支
- 职责是承载 iOS 蓝湖 UI 图适配增强和相关抓取辅助脚本
- 通过合并 `main` 的方式吸收官方更新
- 当前刻意不设置默认上游，避免误把 fork 分支当成同步源直接 `pull`

## 推荐使用方式

### 1. 同步官方源码到本地 `main`

```bash
git fetch upstream
git checkout main
git pull --ff-only
```

如果需要从别的分支上直接把 `main` 强制收敛到上游：

```bash
git fetch upstream
git branch -f main upstream/main
```

### 2. 把最新主线带进 `lanhu-mcp-ios`

```bash
git checkout lanhu-mcp-ios
git merge main
```

### 3. 推送 `main` 到自己的 fork

```bash
git checkout main
git push gitee-fork main
git push github-fork main
```

### 4. 推送 iOS 定制分支到自己的 fork

```bash
git checkout lanhu-mcp-ios
git push gitee-fork lanhu-mcp-ios
git push github-fork lanhu-mcp-ios
```

### 5. 如果 fork 上同名分支已经分叉，但本地才是权威状态

```bash
git checkout lanhu-mcp-ios
git push gitee-fork lanhu-mcp-ios --force-with-lease
git push github-fork lanhu-mcp-ios --force-with-lease
```

## 注意事项

1. `upstream` 只作为读取官方更新的来源，不要把自定义改动直接推上去。
2. `main` 是官方镜像分支，不要在上面直接做 iOS 定制开发。
3. `lanhu-mcp-ios` 的标准更新方式是 `merge main`，不是对 fork 分支直接 `pull`。
4. 如果某个 fork 上的 `lanhu-mcp-ios` 被别处改写过，先确认本地状态后再用 `--force-with-lease` 修正远端。
